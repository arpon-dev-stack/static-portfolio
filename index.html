I use HTML to structure web pages and bring ideas to life on the browser. It’s the foundation of every website I build, helping me create clean and accessible layouts that work across devices.

CSS allows me to design responsive and visually appealing interfaces. I use Flexbox, Grid, and modern styling techniques to ensure that every layout looks great on any screen size.

Node.js allows me to create powerful backend systems, manage APIs, and connect applications to databases. I often pair Node.js with MongoDB for NoSQL flexibility or PostgreSQL for structured, relational data management. This combination helps me build full-stack applications that are both scalable and efficient.

React is my go-to library for building interactive and dynamic user interfaces. I use components, hooks, and React Router to develop fast and modern single-page applications.

Git is my version control tool for managing changes and tracking progress in projects. It allows me to maintain clean code history and experiment safely with new features.

GitHub is my platform for hosting code, showcasing projects, and collaborating with other developers. It plays a vital role in my workflow by allowing me to share and maintain my repositories publicly.

In 2024, I started my web development journey with HTML during my free time. I practiced building simple pages and gradually learned to organize content semantically for better readability and SEO.

I completed the basics of CSS within a couple of days after starting HTML. It was exciting to see how small style changes could completely transform a webpage. Over time, I improved my understanding of layouts and animations to enhance user experience.

After mastering frontend basics, I shifted to backend development with Node.js. I learned to handle routing, requests, and server logic. To strengthen my backend skills, I explored databases — using MongoDB for dynamic, document-based data and PostgreSQL for relational structures. Understanding both gave me a clear view of how data flows through a full-stack application.

To improve my UI skills, I started learning React after grasping the basics of Node.js. I became familiar with key concepts like state, useEffect, useContext, and Axios for data fetching. It took time to understand the component-based mindset, but creating projects helped me master it.

Once I started building multiple projects, I realized the importance of version control. Learning commands like commit, branch, and merge helped me understand how to manage my code efficiently.

After understanding Git, I began using GitHub for collaborative coding and portfolio hosting. I learned how to push commits, open pull requests, and manage repositories — key skills for teamwork in real-world development.
